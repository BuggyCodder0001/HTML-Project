HTML FORMS

Forms in HTML are used to collect user input. A form can contain different input elements like text fields, radio buttons, checkboxes, etc. The form data can then be sent to a server for processing.

Form Elements

  1. <form> : The container element that defines a form.
  2. <input> : Used for user input (text, password, email, checkbox, radio, etc.).
  3. <label> : Defines a label for form elements to improve accessibility.
  4. <textarea> : Multi-line text input field.
  5. <select> : Drop-down list.
  6. <option> : Defines an item inside a drop-down list.
  7. <button> : Clickable button (submit, reset, or custom).
  8. <fieldset> : Groups related form elements together.
  9. <legend> : Caption for a <fieldset>.
  10. <datalist> : Provides auto-complete options for an <input>.
  11. <output> : Displays the result of a calculation or user action.

Basic HTML Form


        <form action="/submit" method="post">
        <label for="name">Name:</label>
        <input type="text" id="name" name="name">
        <button type="submit">Submit</button>
        </form>

The <input> Element: The Box for Your Stuff

This is the most common form tag. It's a self-closing tag that creates an input field. Its behavior changes based on its type attribute.

The most basic type is text.


            <form>
                <input type="text">
            </form>

The Need for Meaning - The<label>

The Problem: We have a box, but the user has no idea what they are supposed to type into it. Is it for a name? An email? A search query? The box is meaningless without a description.

The Solution: We need to add a descriptive piece of text. The correct HTML tag for this is the 


        <label>First Name</label>
        <input type="text">

Result: This is better! Now the user sees "First Name:" next to the box and knows what to type. But the <label> and the <input> are still two completely separate, unrelated things. The browser doesn't know they belong together

Label and Input Connection

  • Use id in the <input> to give it a unique name.
  • Use for in the <label> with the same id.
  • Clicking the label activates the input (Usability).
  • Screen readers announce the correct label (Accessibility). 
            
            <label for="firstName">First Name:</label>
            <input type="text" id="firstName">

The Need for Submission - The <form> and submit

  • Problem: We have fields, but no way to submit the information.
  • Solution:
    1. Wrap all fields inside a <form> tag (acts as container for submission).
    2. Add a submit button using <input type="submit">.
            
            <form>
            <label for="firstName">First Name:</label>
            <input type="text" id="firstName">

            <input type="submit">
            </form>

The Need for Data Identification - The name Attribute

  • Problem: When form is submitted, the server needs labels (keys) to identify data. id works only within the page, not for submission.
  • Solution: Use the name attribute as the "data label" or "key" for submitted values. 
            
            <form>
            <label for="firstName">First Name:</label>
            <input type="text" id="firstName" name="firstName">

            <input type="submit" value="Go">
            </form>

The Need for "Select One" - Radio Buttons

  • Problem: We need to ask a question where the user can choose only one option (e.g., gender, T-shirt size). A text box allows inconsistent answers.
  • Solution: Use <input type="radio">. Selecting one option automatically de-selects others.
  • Rule: All radio buttons in one group must share the same name attribute. The name works as the group identifier. 
            
            <form>
            <p>Choose T-shirt size:</p>

            <input type="radio" id="small" name="size" value="Small">
            <label for="small">Small</label><br>

            <input type="radio" id="medium" name="size" value="Medium">
            <label for="medium">Medium</label><br>

            <input type="radio" id="large" name="size" value="Large">
            <label for="large">Large</label><br>

            <input type="submit" value="Submit">
            </form>

Let's break down the new attributes:

  • type="radio" → Creates the circular radio button.
  • name="shirtSize" → Groups radio buttons together. Same name = only one can be selected.
  • id="sizeS" → Each radio needs a unique id so its label can link correctly.
  • value="small" → Actual data sent to the server if this option is chosen. Example: shirtSize=small.

The Need for "Select Many" - Checkboxes

  • Problem: Sometimes users should be able to choose multiple options (e.g., pizza toppings). Radio buttons only allow one.
  • Solution: Use <input type="checkbox"> which allows multiple selections.
  • Rule: Checkboxes from the same question should share the same name so the server knows all selected values belong to one category. 
            
            <form>
            <label>Pizza Toppings:</label><br>

            <input type="checkbox" id="toppingPep" name="toppings" value="pepperoni">
            <label for="toppingPep">Pepperoni</label><br>

            <input type="checkbox" id="toppingMush" name="toppings" value="mushrooms">
            <label for="toppingMush">Mushrooms</label><br>

            <input type="checkbox" id="toppingOni" name="toppings" value="onions">
            <label for="toppingOni">Onions</label><br>

            <input type="submit" value="Order">
            </form>

The Need for Long-Form Text - <textarea>

  • Problem: <input type="text"> is good for single-line input, but not for long text like comments or addresses (text becomes unreadable).
  • Solution: Use <textarea>, a multi-line text input element.
  • Note: Unlike <input>, <textarea> is a container tag (needs opening and closing tag) and connects to a label using the same for and id pattern. 
            
            <form>
            <label for="comments">Any special instructions?</label><br>
            <textarea id="comments" name="comments" rows="4" cols="50"></textarea><br><br>
            <input type="submit" value="Submit">
            </form>

The Need for Many Options - The <select> Dropdown

  • Problem: Radio buttons work for a few options, but not for long lists (e.g., 200 countries). A long list of radios would make the page messy.
  • Solution: Use a <select> element (dropdown menu) with multiple <option> tags. It hides all options until the user clicks.
  • Note: <label> connects to <select> using for and id. 
            
            <form>
            <label for="country">Country:</label><br>
            <select id="country" name="country">
                <option value="">--Please choose an option--</option>
                <option value="in">India</option>
                <option value="us">USA</option>
                <option value="uk">United Kingdom</option>
                <option value="au">Australia</option>
            </select><br><br>

            <input type="submit" value="Submit">
            </form>
